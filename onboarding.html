架构决策记录（ADR）

3 个核心决策 · 由代码库扫描 + LLM 自动生成 → 人工 review

ADR-001: 业务服务采用微服务架构

状态: Accepted 日期: 2024-08-12 决策人: 架构组 影响范围: 全业务
背景 (Context)
原单体应用（Java + Spring）代码量超 50 万行，单团队交付周期 3 周起，618 大促扩容成本高。需要支持多团队并行交付 + 弹性扩缩容。
决策 (Decision)
按业务域拆分为 5 个核心微服务（订单 / 商品 / 库存 / 会员 / 营销），使用 Spring Cloud + Nacos 注册中心，Kong 网关统一接入，Kafka 异步解耦。
影响 (Consequences)
✅ 正面：交付周期从 3 周降至 1 周 · 大促弹性扩容 3 倍 · 故障隔离 90%
⚠️ 负面：分布式事务复杂度 ↑ · 链路追踪刚需 · 团队运维成本 ↑

ADR-002: 缓存层选型 Redis 8.0（向量化检索）

状态: Accepted 日期: 2026-03-05 决策人: 基础架构 + 推荐团队 影响范围: 商品/推荐
背景 (Context)
商品推荐场景需要向量检索，原方案 FAISS 独立部署，维护两套基础设施。商品 SKU 数 1000 万+，检索延迟 P95 要求 < 15ms。
决策 (Decision)
升级 Redis 到 8.0 版本，使用原生 RediSearch 向量检索能力，统一基础设施。保留 FAISS 作为离线训练引擎。
影响 (Consequences)
✅ 正面：基础设施 1 套 · 维护成本 -50% · P95 12ms 达标
⚠️ 负面：Redis 内存压力 ↑ 30% · 需评估大 Key 风险

ADR-003: 搜索服务从 ES 7.x 升级到 ES 9.x

状态: Proposed 日期: 2026-06-28 决策人: 搜索团队 影响范围: 商品/搜索
背景 (Context)
ES 7.x 已 EOL（2026-07），新版本向量检索性能提升 40%，需考虑升级窗口。
决策 (Decision)
采用蓝绿发布升级 ES 9.x，灰度 5% → 50% → 100%，预计 2 周完成全量切换。
影响 (Consequences)
✅ 预期：检索性能 +40% · 享受官方支持 · 新特性解锁
⚠️ 风险：查询语法部分 deprecated · 需全链路回归
